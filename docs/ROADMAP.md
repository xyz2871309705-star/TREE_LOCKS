## 阶段一：单机版库 （目标：功能完备、可直接集成）

### Iteration 1.1 ✅ 已完成 — 核心协议

> 产出: 锁模式、兼容矩阵、模式转换

- [x] 6 种锁模式定义 (NL/IS/IX/S/SIX/X)
- [x] 兼容矩阵 (6×6 查表，O(1))
- [x] 锁升级路径验证 (15 条合法路径)
- [x] 锁降级路径验证
- [x] 工具函数 (mode_name / strerror)

### Iteration 1.2 ✅ 已完成 — 锁表与等待队列

> 产出: 内存锁表 (单链表)、FIFO 等待队列

- [x] 节点锁状态管理 (单链表，`next` 指针已预留哈希表升级位)
- [x] 授权列表 (grant_list) 与租约过期字段 (lease_expire_at，为 1.6 预埋)
- [x] FIFO 等待队列 + 条件变量唤醒
- [x] 冲突检查 (兼容矩阵 + 同客户端跳过 + 过期跳过)
- [ ] 哈希表升级 (链表 → uthash，O(n) → O(1)) — **推迟至 1.7**

> 当前锁表查找为 O(n) 链表遍历；`treelock_node_t.next` 字段已预留，平滑升级路径明确。

### Iteration 1.3 ✅ 已完成 — 客户端 API

> 产出: 线程安全的 lock/unlock/try_lock

- [x] 生命周期管理 (create/destroy)
- [x] 阻塞锁 (treelock_lock)
- [x] 超时锁 (treelock_try_lock)
- [x] 锁升级/降级 (escalate/downgrade) — 原地更新模式，避免双重 entry
- [x] 同客户端重入加锁 (引用计数 ref_count)
- [x] 查询接口 (get_mode / query_node)

### Iteration 1.4 ✅ 已完成 — 基础工程

- [x] 协议正确性测试 (19 个用例，覆盖兼容矩阵/升级降级/错误码)
- [x] 并发压力测试 (10 个用例，覆盖多线程 lock/unlock/escalate/downgrade)

- [x] 开发者文档 (DEVELOPER.md)

### Iteration 1.5 ✅ 已完成 — 树结构管理

> 产出: JSON 树加载、路径加锁、协议自动校验、24 个树模块单元测试

**实际实现（超计划完成）**:

| 任务 | 说明 |
|------|------|
| JSON 文件加载 | 支持扁平 `[{"id":...}]` 和嵌套 `{"tree":...}` 两种格式 |
| 路径加锁 API | `treelock_lock_path(tl, "/a/b", X)` 自动沿路径逐级加锁 |
| 路径解锁 API | `treelock_unlock_path(tl, "/a/b")` 自底向上释放 |
| 编程式注册 | `treelock_register_node(tl, id, parent, label)` 手动构建树 |

| 向后兼容 | 树未加载时协议校验自动跳过，现有代码不受影响 |
| 模块化架构 | `libtreelock_tree.a` 独立模块，依赖 `treelock_core` + `cjson` |

**新增文件**:
- `modules/treelock_tree/` — 完整模块 (5 源文件 + 1 内部头文件)
- `modules/cjson/` — cJSON v1.7.18 解析器 (MIT)
- `tests/test_tree.cc` — 24 个 GTest 单元测试

**验收**: ✅ `treelock_lock(tl, child, X)` 但未持有 parent 的 IX → 返回 `TREELOCK_ERR_PROTOCOL`

## 迭代优先级排序

按优先级从高到低排列接下来的工作：

| 优先级 | 迭代 | 说明 |
|--------|------|------|
| 🔴 P0 | **1.7** 哈希表升级 + 性能基线 | O(n) 链表查找是当前最大性能瓶颈；uthash 升级工作量小收益大 |
| 🔴 P0 | **1.7** 内存泄漏检测 | valgrind/ASan 审计，确保零泄漏基线 |
| 🟡 P1 | **1.6** 租约机制 | 锁表已预埋 lease 字段，为分布式做准备，单机版也实用 |
| 🟡 P1 | **1.7** 基准测试 + 压力测试 | 建立性能数字基线 |
| 🟢 P2 | **1.8** API 完善 | 批量锁、条件锁、dump 等增强易用性 |
| 🟢 P2 | **2.1** ZK 集成 | 进入分布式阶段 |
| ⚪ P3 | 阶段三各迭代 | 自研服务版，工作量大 |
